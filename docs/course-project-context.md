# Course Project Context

## Origin

This repository is based on an `EEE 416` course project focused on IoT-based patient monitoring. The original work combined:

- patient vital-sign monitoring
- ambient safety and environment monitoring
- a secured medicine box with access logging

## Current status

The prototype itself was implemented and tested during the course project. However, not all of the original details have been preserved over time. In particular, this repository does not claim to be a full archival copy of:

- every hardware integration detail
- all experimental measurements and logs
- the exact final project report wording

## What this repository is

This codebase is a professional software reconstruction of the system architecture and operational flow. It makes the project easier to run, understand, extend, and publish on GitHub by adding:

- a typed REST API
- persistent device, telemetry, alert, and notification models
- a live monitoring dashboard
- a simulator for repeatable demos
- tests, CI, and deployment-ready project structure

## Scope note

This repository should be understood as a polished implementation of the `EEE 416` project concept and tested prototype, updated for maintainability and public sharing.
